# Supabase Project Setup Guide

To enable the custom CMS, Admin Dashboard, and Analytics Tracking on your Vercel-hosted website, you need to set up a free project on [Supabase](https://supabase.com/).

Follow these simple steps:

---

## 1. Create a Supabase Project
1. Log in to [Supabase](https://supabase.com/) and create a new project.
2. Choose a project name, database password, and select your nearest region.
3. Keep the tab open to copy your API keys later.

---

## 2. Run SQL Setup Script
1. In your Supabase Dashboard, click on **SQL Editor** in the left menu.
2. Click **New Query**.
3. Paste the following SQL script in the editor and click **Run** (at the bottom right):

```sql
-- 1. Create site_content table for general text settings
CREATE TABLE site_content (
  key TEXT PRIMARY KEY,
  value JSONB NOT NULL,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- Enable Row Level Security (RLS)
ALTER TABLE site_content ENABLE ROW LEVEL SECURITY;

-- Allow public read access to site_content
CREATE POLICY "Allow public read access" ON site_content
  FOR SELECT USING (true);

-- Allow authenticated admins to insert/update/delete site_content
CREATE POLICY "Allow admin writes" ON site_content
  FOR ALL TO authenticated USING (true);


-- 2. Create experiences table for portfolio cards
CREATE TABLE experiences (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  role TEXT NOT NULL,
  company TEXT NOT NULL,
  location TEXT,
  date_range TEXT NOT NULL,
  category TEXT NOT NULL,
  description TEXT,
  details JSONB NOT NULL,
  impact_metrics JSONB NOT NULL,
  order_index INTEGER NOT NULL,
  budget_usd NUMERIC DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- Enable Row Level Security (RLS)
ALTER TABLE experiences ENABLE ROW LEVEL SECURITY;

-- Allow public read access to experiences
CREATE POLICY "Allow public read access" ON experiences
  FOR SELECT USING (true);

-- Allow authenticated admins to write experiences
CREATE POLICY "Allow admin writes" ON experiences
  FOR ALL TO authenticated USING (true);


-- 3. Create analytics_events table for tracking visitor events
CREATE TABLE analytics_events (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  session_id TEXT NOT NULL,
  event_type TEXT NOT NULL,
  event_label TEXT,
  duration INTEGER DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- Enable Row Level Security (RLS)
ALTER TABLE analytics_events ENABLE ROW LEVEL SECURITY;

-- Allow public inserts to analytics_events (so visitors can record views/durations)
CREATE POLICY "Allow public inserts" ON analytics_events
  FOR INSERT WITH CHECK (true);

-- Allow authenticated admins to view/manage analytics_events
CREATE POLICY "Allow admin reads" ON analytics_events
  FOR ALL TO authenticated USING (true);
```

### If you already ran the first setup query:
If you already set up the project earlier, you only need to run this single line to add the budget column:
```sql
ALTER TABLE experiences ADD COLUMN budget_usd NUMERIC DEFAULT 0;
```

## 3. Create Storage Buckets
To upload and store your CV PDF, Portfolio PDF, and brand logos, create a storage bucket:
1. Click **Storage** in the left sidebar menu of Supabase.
2. Click **New Bucket**.
3. Name the bucket exactly **`assets`**.
4. Toggle the **Public bucket** switch to **ON** (so visitors can view/download your files).
5. Click **Create bucket**.

### Set Storage Security Policies:
To allow the Admin Console to upload files:
1. Select the `assets` bucket.
2. Click **Policies** on the left menu (under Storage).
3. Find the policy area and click **New Policy** -> **For full customization**.
4. Create an **Upload / Delete / Update** policy with:
   * **Allowed Operations:** Select `INSERT`, `UPDATE`, `DELETE`, `SELECT` (Check all boxes).
   * **Target Roles:** Select `authenticated`.
   * **Using expression / check expression:** Enter `true`.
   * Save the policy. (This allows you to overwrite PDFs/logos from the Admin page after logging in).

---

## 4. Create an Admin User
To log into your admin dashboard:
1. Go to **Authentication** in the left menu.
2. Click **Add User** -> **Create User**.
3. Enter the email address and password you want to use for the Admin Dashboard.
4. Toggle **Auto-confirm User** to **ON** (so you don't need email verification).
5. Click **Save**.

---

## 5. Add Keys to your Project & Vercel
Create a `.env` file in the root of your local workspace `/Users/amrsamiredris/Desktop/amrsamirsite/` to test locally:

```env
VITE_SUPABASE_URL=your_project_supabase_url
VITE_SUPABASE_ANON_KEY=your_project_anon_key
```

When you deploy to Vercel, copy these exact environment variables into Vercel's **Environment Variables** settings tab so your live site can connect to Supabase.
